VCS Implementationn Using Github

Create repository

Click on new repository.
Enter details.

Clone repository

Copy your repository path.
Move to directory where you want to clone your repository and then paste path in clone command.

Local repository

Now move to your local repository and start working in it.

Add file to local repository

Create a simple html file named "index.html" in your local repository.
" git add . " is used to add all changes in the working directory to the staging area.

Commit changes on local repository

Using the "git commit" command only saves a new commit object in the local Git repository.
You can also write commit message.

Push changes to remote repository

Push all the changes to remote repository.
Now you can see your index file is being added on remote repository.

Edit file on remote repository

Click on "pen" shaped icon to edit your file online.

Commit changes on remote repository

Index file updated.

Get data from remote repository to local one

Pull command is used to get changes from remote repository to your local repository.

Resolve Merge Conflicts

A merge conflict is an event that occurs when Git is unable to automatically resolve differences in code between two commits.
A conflict arises when two separate branches have made edits to the same line in a file. Here you can see when we pull from github, conflict arises.
Here you can see, your editor is giving options for choosing what data you want to keep. I'm choosing incoming changes.
Successfully pushed.

To check list of all the commits made to repository

The git log command shows a list of all the commits made to a repository.

Branching in Github

When you create a repository with content on GitHub, GitHub creates the repository with a single branch.
This first branch in the repository is the default branch.
Following checkout command is used to create and switch between branches.
Adding test file in test branch.
Here you can see all the branches.

Merge different branches

The "merge" command is used to integrate changes from another branch.
Here we are merging test branch into master.
To merge, first go into branch in which you want to merge.In merge command, write the name of branch you want to merge.
